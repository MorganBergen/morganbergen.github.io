# Website Roadmap

## Objective

Build a professional portfolio website in 7 days that supports job and research applications. The site should be text-forward, minimal, elegant, and easy to maintain by hand.

## Product Direction

- Tone: precise, calm, professional
- Visual language: terminal-inspired, editorial, mostly text
- Version one scope: no heavy media, no animation dependency, no unnecessary pages
- Primary audience: software engineering employers and undergraduate research mentors
- Primary outcome: a reviewer should understand your background, strengths, and contact information quickly

## Site Map

1. Home
2. About
3. Portfolio
4. Resume
5. Contact

## Page Purpose

### Home

- Short introduction
- Current role and goals
- Brief positioning statement
- Fast links to resume, GitHub, portfolio, contact

### About

- Background as a senior in computer science with a mathematics minor
- Interests in software engineering and research
- How you approach learning, code quality, and problem solving

### Portfolio

- Two to four selected projects
- Short summary, tech stack, your contribution, and outcome
- Optional links to code or demo

### Resume

- Web-readable summary of experience and skills
- Download link for PDF resume
- Education, projects, coursework, skills

### Contact

- Email
- GitHub
- LinkedIn
- Short note about what roles you are seeking

## UX Rules

- One clear navigation system across all pages
- Layout should feel stable and consistent
- Copy should be concise and easy to scan
- Every page should have one obvious purpose
- Mobile should be treated as first-class, not a later fix

## Design Rules

- Use an SF Mono based font stack, with system mono fallbacks
- Support dark mode and light mode
- Support a small set of palette options without changing layout
- Keep spacing generous and visual noise low
- Use borders, rhythm, and typography instead of imagery

## Suggested Stack

- React for UI
- Plain CSS for version one
- GitHub Pages or Vercel for deployment

## Seven-Day Schedule

### Day 1

- Finalize scope
- Approve site map
- Gather resume, links, and project source material
- Create GitHub milestones and issues

### Day 2

- Finalize wireframe and content structure
- Lock typography, spacing, colors, and theme tokens
- Build the layout shell

### Day 3

- Implement navigation and page containers
- Add dark and light mode
- Add palette switcher

### Day 4

- Write real copy for Home and About
- Add portfolio project entries
- Add resume page structure

### Day 5

- Refine Contact page
- Improve responsive behavior
- Add metadata and polish details

### Day 6

- QA on desktop and mobile
- Fix accessibility, spacing, and visual consistency issues
- Tighten copy

### Day 7

- Final review
- Deploy production version
- Share with a few reviewers
- Collect follow-up notes for version two

## Definition of Done

- All five pages are implemented
- Real content replaces placeholders
- Theme switching works
- Palette switching works
- Mobile and desktop layouts are acceptable
- Resume PDF is accessible
- Contact information is correct
- Site is deployed and publicly reachable

## Risks to Avoid

- Spending too long on visual experimentation
- Adding too many projects or sections
- Delaying content writing until the end
- Overengineering routing, state, or animations in version one
